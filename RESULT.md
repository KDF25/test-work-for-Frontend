# Найденные проблемы

## 1. Качество кода
- Использование `any[]` вместо строгих типов.  
- Дублирование кода: функции `formatDate` и `formatTaskDate`.  
- Неиспользуемые переменные:  
  `userTasks`, `userFilter`, `filteredUserTasks`,  
  `loadUserTasks`, `calculateCompletionRate`, `getCompletionPercentage`.  
- Неиспользуемый `computed`: `taskStats` вычисляется, но не используется.  
- Утечка памяти: event listeners не удаляются при размонтировании компонента.  
- Отсутствие очистки таймеров при отмене удаления.  
- Отсутствие валидации: не проверяются пустые строки при добавлении задачи.

## 2. Архитектура
- Монолитный компонент: вся логика в одном `App.vue`.  
- Нет использования Pinia/Vuex для управления состоянием.  
- Отсутствие типизации: нет интерфейсов для `Task`.  
- Смешанная ответственность: UI, бизнес-логика и работа с данными в одном месте.

## 3. Производительность
- Множественные вычисления в шаблоне: `tasks.filter()` вызывается несколько раз.  
- Нет debounce для частых операций.  
- Синхронные задержки: искусственные задержки в `loadTasks` и `loadUserTasks`.  
- Процент завершения вычисляется inline в шаблоне.

## 4. Безопасность
- XSS-уязвимость: использование внешнего SVG без проверки.  
- Отсутствие санитизации пользовательского ввода.  
- Небезопасное использование `Date.now()` для генерации ID.
